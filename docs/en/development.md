# Development

How to work on Ren Browser locally.

## Requirements

- Go 1.26+
- Node.js 22+ and pnpm 11+
- Task (recommended)
- Reticulum config for live mesh tests (optional)

## Clone and run

```sh
git clone https://github.com/Quad4-Software/Ren-Browser.git
cd Ren-Browser
task dev
```

`task dev` runs Wails in development mode with Vite hot reload on port 9245 by default.

## Common tasks

| Task | Purpose |
|------|---------|
| `task dev` | Desktop dev mode |
| `task build` | Production build for current OS |
| `task run` | Run built binary |
| `task check` | Full quality gate (Go + frontend) |
| `task test` | All unit tests |
| `task test:interop` | Live Reticulum tests (needs network) |
| `task build:server` | Headless server binary |
| `task run:server` | Run server locally |
| `task package` | Platform installer or bundle |

## Quality gate

Before you send changes:

```sh
task check
```

`check` runs:

- `gofmt` on Go sources
- `go test ./...`
- gosec security scan
- Brand consistency check
- Frontend typecheck, lint, format check, knip, audit, and vitest

Optional harder Go tests:

```sh
task test:go:race
task test:go:hard
task fuzz:go
```

## Frontend only

```sh
cd frontend
pnpm install
pnpm check
pnpm test
```

Bindings under `frontend/bindings/` are generated by Wails from Go services.

## Go layout

| Path | Role |
|------|------|
| `main_desktop.go` | Desktop entry (Wails window) |
| `main_server.go` | Server entry (HTTP only) |
| `internal/app/` | Browser service exposed to UI |
| `internal/rns/` | Reticulum stack wrapper |
| `internal/nomadnet/` | Page fetch over LXMF. NomadNet nodes are detected from announces, not from NomadNet libraries |
| `internal/micron/` | Micron rendering |
| `internal/store/` | SQLite persistence API |
| `internal/plugins/` | Extension host |
| `frontend/` | Svelte 5 UI |

See [Architecture](architecture.md) for a fuller map.

## Plugin development

```sh
task test:plugins
```

Install your dev plugin into `~/.renbrowser/plugins/` and reload the app.

## Interop tests

```sh
task test:interop
```

Requires a live Reticulum network and the `interop` build tag. Skip for UI-only work.

## Asset probe

Set `REN_BROWSER_ASSET_PROBE=1` when debugging which asset loader (embedded vs disk) is active.

## SPDX headers

New Go files should include:

```go
// SPDX-License-Identifier: MIT
```

Match existing style in neighboring files.
