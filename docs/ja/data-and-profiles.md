# データとプロファイル

Ren Browser はブックマーク、履歴、タブ、設定、ディスカバリーキャッシュをディスクに保存します。このページでは、データの保存場所とプロファイルの仕組みを説明します。

## デフォルトの場所

| 項目 | パス |
|------|------|
| データディレクトリ | `~/.renbrowser/` |
| メインデータベース | `~/.renbrowser/renbrowser.db` |
| プラグイン | `~/.renbrowser/plugins/<id>/` |
| 名前付きプロファイル | `~/.renbrowser/profiles/<name>/renbrowser.db` |
| レガシー状態（移行済み） | `~/.renbrowser/state.json` |

アップグレード後の初回起動時、`state.json` は自動的に SQLite にインポートされます。

## SQLite に保存されるもの

一般的なテーブルとブロブには以下が含まれます：

- 開いているタブとセッション復元状態
- タイムスタンプ付きのブラウジング履歴
- お気に入り
- ブラウザの設定とキーバインド
- キャッシュされたディスカバリーエントリ
- テーマの選択とカスタムテーマデータ

開く際に破損が検出されます。UI がデータベースのリセットを提案することがあります。リセットするとローカルのタブ、履歴、お気に入り、設定が削除されます。

## Reticulum データは別

アイデンティティキーとインターフェース設定は Reticulum ディレクトリ（デフォルトは `~/.reticulum-go/`）に保存されます。Ren Browser はそのパスを読み込みますが、Reticulum のアイデンティティを `~/.renbrowser/` に移動しません。

## 名前付きプロファイル

`--profile NAME` または `REN_BROWSER_PROFILE=NAME` で起動すると以下を使用します：

```
~/.renbrowser/profiles/NAME/renbrowser.db
```

1 つのアカウントで履歴を分けたい場合（仕事とプライベート、またはテスト用）にプロファイルを使用してください。

## インポートとエクスポート

起動時のみ：

- `--export-profile /path/to/backup.json` はプロファイルデータを書き込んで終了する
- `--import-profile /path/to/backup.json` はファイルからマージまたは置換する

環境変数の対応：`REN_BROWSER_EXPORT_PROFILE`、`REN_BROWSER_IMPORT_PROFILE`。

大きなアップグレードの前や新しいマシンに移行する際にエクスポートを使用してください。

## サーバーモードのストレージ

| モード | タブ、履歴、お気に入り |
|--------|---------------------|
| デフォルトサーバー | サーバーの `~/.renbrowser/` にあるサーバーサイドの SQLite |
| `--public-mode` | 各クライアントのブラウザの `localStorage` |

多くのユーザーが 1 つのサーバーインスタンスを共有する場合は公開モードを選択してください。

## テーマのインポートとエクスポート

テーマは設定から JSON としてエクスポートでき、別のインストールでインポートできます。テーマファイルはプロファイル全体ではなく、外観トークンのみです。

## プラグインデータ

`storage.plugin` 権限を持つ拡張機能は、プラグイン ID でキー付けされた分離ストレージを取得します。プラグインをアンインストールしても、そのフォルダーが常に削除されるわけではありません。クリーンな削除が必要な場合は `~/.renbrowser/plugins/<id>/` を手動で削除してください。

## Android

モバイルビルドはアプリのサンドボックス内で同じ論理レイアウトを使用します。OS のルールによりパスは異なりますが、データベーススキーマはデスクトップと一致します。

## バックアップチェックリスト

1. Ren Browser を停止する
2. `~/.renbrowser/renbrowser.db`（またはプロファイルパス）をコピーする
3. メッシュのアイデンティティも必要な場合は `~/.reticulum-go/` をコピーする
4. 拡張機能を使用している場合は `~/.renbrowser/plugins/` をコピーする

次回起動前にファイルを元の場所に戻すことで復元できます。
