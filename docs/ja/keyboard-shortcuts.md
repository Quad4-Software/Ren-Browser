# キーボードショートカット

デスクトップのデフォルトキーバインドです。**設定 → キーバインド** で変更できます。

`mod` は Windows および Linux では **Ctrl**、macOS では **Cmd** を意味します。

## デフォルトのバインド

| アクション | コード | 説明 |
|-----------|-------|------|
| アドレスバーにフォーカス | `mod+l` | キーボードフォーカスを URL フィールドに移動 |
| ページを再読み込み | `mod+r` | アクティブなタブを再読み込み |
| 開発者ツール | `mod+shift+i` | DevTools を開く/閉じる |
| ページ内検索 | `mod+f` | 現在のページに検索バーを開く |
| ディスカバリーパネル | `mod+shift+d` | ディスカバリーサイドバーを開く |
| 設定パネル | `mod+,` | 設定を開く |
| 新しいタブ | `mod+t` | 空白のタブを開く |
| 新しいウィンドウ | `mod+shift+n` | 別のウィンドウを開く（サポートされている場合） |
| タブを閉じる | `mod+w` | アクティブなタブを閉じる |
| フルスクリーン | `f11` | フルスクリーンを切り替え |

## 新しいバインドを記録する

1. **設定 → キーバインド** を開く
2. 変更したいアクションをクリックする
3. 新しいキーコンビネーションを押す
4. 別のアクションとの競合が UI に表示される。保存前に解決してください。

記録中は、レコーダーが新しいコードのみを検出できるよう、他のショートカットは一時停止されます。

## コード構文

バインドは `+` で結合された小文字のコードとして保存されます：

- `mod`：Ctrl または Cmd
- `shift`：Shift
- `alt`：Alt
- 最後のセグメントはキー名（`l`、`r`、`,`、`f11` など）

例：`mod+shift+d` は Linux では Ctrl+Shift+D です。

## 拡張機能のキーバインド

拡張機能は `renbrowser.plugin.json` のオプションの `keybind` フィールドを持つコマンドを提供できます。有効な拡張機能はそのコマンドをホストのコマンドパレットとショートカット処理にマージします。

## Android

Android 上のハードウェアキーボードは、接続されている場合に同じコードルールに従います。タッチ UI ではショートカットは不要です。
