# セキュリティ

Ren Browser は信頼できるシステムとネットワーク向けに設計されています。このページでは、安全な利用、ダウンロードの検証、問題の報告方法についてまとめます。

## 信頼の境界

| 表面 | リスク |
|------|-------|
| デスクトップアプリ | Go バインディングを持つローカル webview。ページコンテンツに Node.js なし。 |
| サーバーモード | オープンな HTTP ポート。ログイン機能なし。 |
| プラグイン | 宣言された権限で実行。WASM プラグインはキャップでサンドボックス化。 |
| メッシュコンテンツ | 任意のネットワークコンテンツと同様に信頼されない。プラグインが `render.unsanitized` を要求しない限り、Micron HTML はサニタイズされる。 |

## サーバーモード

`renbrowser-server` には**認証機能がありません**。インターネットに公開すると以下のリスクがあります：

- 自動スキャン
- Reticulum インターフェースの悪用
- シングルプロセスアプリの過負荷

公開が必要な場合：

1. アクセス制御を持つリバースプロキシの背後に置く
2. 有効な証明書で HTTPS を使用する
3. ファイアウォールまたは VPN ルールでポートを制限する
4. アプリを最新の状態に保つ

プロキシフラグについては [サーバーモード](server-mode.md) を参照してください。

## デスクトッププラグイン

信頼できる人やプロジェクトからの拡張機能のみをインストールしてください。有効化する前に **設定 → 拡張機能** の権限リストを読んでください。

### インストール時のチェック

拡張機能をインストールすると、RenBrowser は以下を表示します：

- 権限ごとのトグル（無効にした権限は実行時に付与されない）
- マニフェストとパッケージファイルからスキャンされたネットワークエンドポイント
- 署名バッジ（未署名、署名済み、信頼できるパブリッシャー、改ざん済み）
- ヒューリスティックセキュリティ評価（未署名＋ネットワーク、危険な権限、JS の不審なパターン）

無効な署名はインストールをブロックします。未署名の拡張機能はリスクを承認すればインストールできます。

### 実行時保護

- 付与された権限は JS の `PluginFetch` と WASM の `http_fetch` に対して強制される
- `network.fetch` が付与されていない場合、WASM ネットワークエクスポートはブロックされる
- プラグインの HTTP リクエストと WASM 処理に対する呼び出しごとの制限により、誤動作する拡張機能によるフリーズリスクを軽減する
- 拡張機能のファイル整合性はインストール後にハッシュ化される。外部からの改ざんにより再有効化するまで拡張機能が無効化される
- アプリ外でのユーザー信頼パブリッシャーリストの変更は、データベースバックアップのダイジェストで検出される

プラグインの HTTP トラフィックは DevTools が開いている場合に **開発者ツール → ネットワーク** に表示されます。

署名、ロケール、マニフェストフィールドについては [拡張機能](extensions.md) を参照してください。

## ダウンロードの検証

公式ビルドは [GitHub Releases](https://github.com/Quad4-Software/Ren-Browser/releases) と GitHub Actions CI から提供されます。

各リリースには `SHA256SUMS.txt` が含まれているはずです。ファイルを確認してください：

```sh
sha256sum -c SHA256SUMS.txt
```

Docker の場合、ビルドを信頼した後はダイジェストによるピン留め（`@sha256:...`）を優先してください。GHCR 上のイメージには Docker Buildx からのビルドプロビナンスと SBOM が含まれています。

バイナリが公開されたチェックサムと一致しない場合は、信頼できないものとして扱ってください。

## WASM のサブリソース整合性

Micron パーサーの WebAssembly とその `wasm_exec.js` コンパニオンは実行前に SHA-384 SRI でチェックされます。ハッシュの不一致はコードをブロックし、エラーを表示します。

## 保存データ

- アプリケーション状態：`~/.renbrowser/` 以下の SQLite
- Reticulum 鍵：Reticulum 設定ディレクトリ
- サーバー公開モード：一部のデータは各クライアントのブラウザの `localStorage` のみ

マシンが共有または携帯用の場合は、OS レベルでディスクを暗号化してください。

## 脆弱性の報告

未修正のセキュリティバグを公開の GitHub Issue として開かないでください。

**推奨する連絡先：**

1. LXMF：`f489752fbef161c64d65e385a4e9fc74`

バージョン、プラットフォーム、再現手順、影響範囲を含めてください。

## CI とサプライチェーン（概要）

GitHub Actions はテスト、gosec、Trivy スキャン、CodeQL をスケジュールで実行します。サードパーティの Action はワークフローでコミット SHA にピン留めされています。
