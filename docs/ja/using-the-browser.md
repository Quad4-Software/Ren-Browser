# ブラウザの使い方

このページでは、タブ、アドレスバー、履歴、お気に入り、ページ表示といった日常的な利用方法について説明します。

## ウィンドウのレイアウト

主なエリアは以下のとおりです：

- 上部の **タブバー**。タブをドラッグしたり、固定したり、分割ビューで開いたりできます。
- デスティネーションと組み込みスキーム用の **アドレスバー**。
- Micron ページがレンダリングされる **コンテンツエリア**。
- ディスカバリー、履歴、設定、DevTools、拡張パネル用の **サイドパネル**。

小さい画面では、モバイルナビゲーションバーがデスクトップの一部の要素を置き換えます。

## タブ

- **新しいタブ**：デフォルトのショートカット `Ctrl+T` / `Cmd+T`（[キーボードショートカット](keyboard-shortcuts.md)を参照）
- **タブを閉じる**：`Ctrl+W` / `Cmd+W`
- **セッションの復元**：開いているタブはローカルデータベースに保存され、次回起動時に復元される

固定されたタブはタブストリップの先頭に残ります。分割ビューで 2 つのタブを並べて表示できます。

## アドレスバー

NomadNet のデスティネーションを入力するか、組み込みスキームを使用します：

| 入力 | 結果 |
|------|------|
| 32 文字の 16 進数ハッシュ | `hash:/page/index.mu` を開く |
| 完全な NomadNet URL | 入力したまま開く |
| `about:` | バージョンとパスを含む About ページ |
| `license:` | ライセンステキスト |
| `editor:` | Micron エディタ |
| `settings` | 設定を開く（UI ルーティング経由） |

Enter を押してナビゲートします。`Ctrl+L` / `Cmd+L` でバーにフォーカスできます。

## リンクをたどる

ページ上の Micron リンクは現在のデスティネーションを基準に解決されます。内部の `about:`、`license:`、`editor:` リンクは通常のナビゲーションのように動作します。

外部のメッシュリンクは Reticulum のデスティネーション構文を使用します。リンクが失敗した場合は、ターゲットノードが到達可能かどうかを確認してください。

## 履歴

サイドバーまたはキーバインドから履歴パネルを開きます。以下のことができます：

- タイトルまたは URL で検索する
- 日付別にグループ化されたエントリを表示する
- 過去のページを現在のタブまたは新しいタブで開く

履歴はローカルの SQLite（デスクトップ）に保存されます（サーバーの公開モードを使用している場合を除く）。

## お気に入り

ページのコンテキストメニューまたはディスカバリーから、よく訪れるノードを保存します。お気に入りは履歴とタブと同じストアと同期します。

## ページ内検索

`Ctrl+F` / `Cmd+F` を押して現在の Micron ページのテキストを検索します。マッチした箇所がコンテンツビューアーでハイライト表示されます。

## 開発者ツール

`Ctrl+Shift+I` / `Cmd+Shift+I` を押して DevTools を開きます。以下の用途に便利です：

- レンダリングタイミングの検査
- 生のページソースの表示
- devtools エントリを提供する拡張パネルのデバッグ

## ダウンロード

ページまたはアクションがファイルを提供する場合、ダウンロードメニューにアイテムが表示されます。デスクトップではパスが OS のダウンロード規則に従います。

## ページエラー

ページが読み込めない場合、短いメッセージと共にエラー状態が表示されます。よくある原因：

- デスティネーションに到達できない
- 無効な Micron コンテンツ
- Reticulum が接続されていない

接続を修正した後、**再読み込み**（`Ctrl+R` / `Cmd+R`）を使用してください。

## Micron エディタ

`editor:` を開いて組み込みエディタで Micron を作成します。NomadNet ノードに公開する前のローカル下書きに使用してください。
