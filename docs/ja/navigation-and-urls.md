# ナビゲーションと URL

Ren Browser のアドレスバーはいくつかの URL 形式を受け付けます。このページではそれらをリストし、正規化の仕組みを説明します。

## NomadNet のデスティネーション

完全な NomadNet URL は次のようになります：

```
<32-hex-chars>:/page/path.mu
```

例：

```
a1b2c3d4e5f6789012345678abcdef01:/page/index.mu
```

ハッシュは 16 進数の Reticulum デスティネーション ID です。`:/` の後のパスは NomadNet ノード上のファイルです。

## 短縮ハッシュ

32 文字の 16 進数文字列のみを入力すると、Ren Browser は次のように展開します：

```
<hash>:/page/index.mu
```

これは通常の NomadNet ホームページパスと一致します。

## ハッシュなしのパス

現在のコンテキストにすでにデスティネーションがある場合、`/page/` で始まるパスがそのノードを基準に解決されることがあります。コールドナビゲーションでは、完全な `hash:/page/...` 形式を使用することを推奨します。

## 組み込みスキーム

これらのスキームはアプリ内で処理されます。メッシュは使用しません。

| スキーム | エイリアス | 説明 |
|--------|---------|-----|
| `about:` | `about` | バージョン、ビルド、Reticulum 設定パス、データディレクトリ |
| `license:` | `license` | プロジェクトライセンス（MIT） |
| `editor:` | `editor` | Micron ソースエディタ |

マッチングは大文字小文字を区別しません。末尾のスペースはトリムされます。

## 拡張 URL スキーム

インストール済みの拡張機能は `renbrowser.plugin.json` でカスタムスキームを登録できます。例えば hello 拡張機能は `hello:` を登録します。[拡張機能](extensions.md)を参照してください。

## 設定と内部ルート

UI はパネルに内部ルートを使用することがあります。アドレスバーはメッシュと組み込みスキームに焦点を当てています。サイドバーボタンまたは `Ctrl+,` / `Cmd+,` で **設定** を開きます。

## タブのタイトル

タブのタイトルは以下から取得されます：

1. ノードがタイトルを提供する場合のページメタデータ
2. ハッシュが既知のノードと一致する場合のディスカバリー表示名
3. フォールバックとして短縮されたハッシュまたはパス

## 履歴エントリ

コンテンツを読み込むナビゲーションは、URL、タイトル、デスティネーションハッシュ、タイムスタンプを含む履歴行を作成できます。`about:` のような組み込みページも含まれます。

## Micron ページのリンク解決

レンダリングされたページのリンクをクリックすると：

- `about:`、`license:`、`editor:` はローカルで開く
- 絶対メッシュ URL は直接ナビゲートする
- 相対パスは現在のページデスティネーションと組み合わされる

## 正規化ルール（まとめ）

| 入力 | 正規化された URL |
|------|----------------|
| `about` | `about:` |
| `license` | `license:` |
| `editor` | `editor:` |
| `abcdef...`（32 文字の 16 進数） | `abcdef...:/page/index.mu` |
| 完全な `hash:/page/foo.mu` | 変更なし |
