# Настройка Reticulum

Ren Browser использует [Reticulum](https://reticulum.network/) через стек `quad4/reticulum-go`. Ниже : что ожидает приложение и как исправить типичные проблемы mesh.

## Расположение конфига по умолчанию

| Параметр | Путь по умолчанию |
|----------|-------------------|
| Каталог конфига Reticulum | `~/.reticulum-go/` |
| Флаг переопределения | `--config /path/to/config` |
| Переменные окружения | `REN_BROWSER_CONFIG` или `RETICULUM_CONFIG` |

Точный набор файлов зависит от вашей установки Reticulum или reticulum-go. Ren Browser поднимает стек при запуске и подхватывает изменения интерфейсов из **Настроек**.

## Что происходит при старте

1. Ren Browser загружает конфиг Reticulum
2. Поднимаются интерфейсы (UDP, TCP, RNode и другие из конфига)
3. Анонсы узлов NomadNet появляются в **Обнаружении**
4. Запросы страниц уходят по LXMF и Reticulum к узлам с размещёнными страницами Micron

При сбое смотрите лог в терминале (настольная версия) или логи контейнера (сервер). Приложение продолжит работу: доступны `about:` и **Настройки**.

## Интерфейсы в настройках

В разделе Reticulum можно:

- видеть активные интерфейсы
- смотреть счётчики переданных и принятых байт
- править конфиг и применять hot reload без полного перезапуска

Удобно при добавлении интерфейса или смене ключей.

## Подключение к mesh

Нужен хотя бы один путь к другим узлам Reticulum. Частые варианты:

- **Локальный UDP или TCP** в LAN с другими узлами
- **RNode** или другое радиооборудование
- **Описания интерфейсов** на известные узлы или хабы

Подробности Reticulum : в [руководстве](https://reticulum.network/manual/). Здесь только то, что нужно браузеру.

## Адреса NomadNet

Страницы живут на назначениях Reticulum. В адресной строке можно указать:

- полный путь, например `abcdef0123456789abcdef0123456789:/page/index.mu`
- только 32-символьный hex-хеш (Ren Browser добавит `:/page/index.mu`)

Страницы в формате Micron. Ren Browser рендерит их встроенным конвейером.

## Пустое Обнаружение

Проверьте по порядку:

1. Reticulum запущен внутри Ren Browser (интерфейсы видны в Настройках)
2. Интерфейсы совпадают с тем, как настроены пиры в mesh
3. Подождите немного после подключения. Анонсы приходят не мгновенно
4. Убедитесь, что вы в той же логической сети, что и ожидаемые узлы

## Таймаут или ошибка загрузки страницы

1. Проверьте хеш назначения
2. Убедитесь, что есть маршрут до узла (не только запись в Обнаружении)
3. Откройте другой известный узел из Обнаружения
4. Смотрите devtools или логи на ошибки LXMF и транспорта

## Сервер и Docker

При запуске Docker-образа `renbrowser` смонтируйте каталог Reticulum с хоста и запускайте от имени вашего пользователя, чтобы контейнер без root мог читать ключи и писать mesh-хранилище:

```sh
--user "$(id -u):$(id -g)" \
-e HOME=/data \
-v "$HOME/.reticulum-go:/data/.reticulum-go" \
-v "$HOME/.renbrowser:/data/.renbrowser" \
-e REN_BROWSER_CONFIG=/data/.reticulum-go/config
```

Не монтируйте конфиг только для чтения: Reticulum должен обновлять хранилище рядом с файлом конфигурации.
