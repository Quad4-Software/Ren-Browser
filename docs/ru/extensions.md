# Расширения

Плагины добавляют URL-схемы, боковые панели, команды, темы, страницы настроек и вкладки DevTools.

## Установка

### Через настройки

1. **Настройки → Расширения**
2. **Установить расширение**, затем **.zip**, **папку** или **собранный .wasm-модуль**
3. Просмотрите предпросмотр установки:
   - запрашиваемые разрешения (отдельные можно отключить до установки)
   - внешние URL, с которыми может связываться расширение (сканирование манифеста и файлов пакета)
   - статус подписи издателя (без подписи, подписано, доверенный издатель, недействительная подпись)
   - предупреждения оценки безопасности
   - встроенные языки интерфейса (при наличии `locales/*.json`)
4. Подтвердите и включите расширение

Расширения с `network.fetch` показывают диалог с обнаруженными endpoint. Список URL остаётся видимым, даже если вы отключите `network.fetch` перед установкой.

### Вручную

Распакуйте в:

```
~/.renbrowser/plugins/<id>/
```

В каталоге должен быть `renbrowser.plugin.json`. `id` в манифесте должен совпадать с именем папки.

## Примеры

В репозитории: `extensions/hello-extension/` (схема `hello:`, панель **Hello**, команда **Say hello**).

`extensions/micron-translator/` переводит страницы Micron (`.mu`) через Google Translate или LibreTranslate. Команды: **Translate Micron page** (`mod+shift+t`) и **Restore original** (`mod+shift+r`).

## Файл манифеста

Имя: `renbrowser.plugin.json`

Обязательные поля:

| Поле | Назначение |
|------|------------|
| `manifestVersion` | Сейчас `1` |
| `id` | Уникальный id (3–128 символов: буквы, цифры, `.`, `-`) |
| `name` | Отображаемое имя |
| `version` | Строка semver |
| `main` | Точка входа фронтенда (опционально) |
| `permissions` | Список возможностей |

Опционально: `description`, `author`, `license`, `engines`, `backend`, `network`, `contributes`.

### Ограничение версии

```json
"engines": { "renbrowser": ">=0.1.0" }
```

Хост не загрузит плагин, если версия приложения слишком старая.

### Сетевые endpoint

Расширения с `network.fetch` должны объявлять контактируемые хосты или URL:

```json
"network": {
  "endpoints": [
    "https://api.example.com/",
    "URL сервиса, задаваемый пользователем"
  ]
}
```

При установке RenBrowser также сканирует `.js`, `.go`, `.wasm` и другие файлы пакета на `http`/`https` URL.

### Вклады (contributes)

| Тип | Назначение |
|------|------------|
| `urlSchemes` | Свои URL-схемы |
| `panels` | Панели в боковой панели и др. |
| `commands` | Команды и горячие клавиши |
| `themes` | JSON тем |
| `settings` | Подстраницы настроек |
| `devtools` | Вкладки DevTools |
| `renderers` | Рендереры для MIME или расширений |

## Разрешения

| Разрешение | Позволяет |
|------------|-----------|
| `storage.plugin` | Локальное хранилище плагина |
| `navigation.read` | Читать URL и вкладки |
| `navigation.write` | Инициировать навигацию |
| `network.fetch` | Сетевые запросы через разрешённые API |
| `events.emit` | Отправлять события хоста |
| `events.subscribe` | Подписываться на события |
| `devtools.network` | Доп. сеть в DevTools |
| `render.unsanitized` | Меньше санитизации HTML (опасно) |

Хост проверяет разрешения в runtime. Отключённые при установке права не передаются в JS `ctx.network.fetch` и WASM `http_fetch`.

## Подписи издателя

Расширения могут содержать подпись Ed25519 в `renbrowser.plugin.rsg` (совместимо с Reticulum `rnid`). Недействительная подпись блокирует установку.

Значки в предпросмотре и списке:

| Значок | Значение |
|--------|----------|
| Без подписи | Нет файла подписи |
| Подписано | Действительная идентичность Reticulum |
| Доверенное | Издатель из списка доверия |
| Изменено | Файлы изменены вне RenBrowser (отключено до повторного включения) |

При установке можно выбрать **Доверять этой идентичности издателя**. Пользовательский список: `~/.renbrowser/trusted_publishers.json`, защищён digest в базе профиля.

Подпись: `build/scripts/sign-extension.sh` (нужен Python `rnid`).

## Переводы UI расширения

Строки интерфейса в `locales/<code>.json`. В манифесте заголовки могут использовать `%ключ.путь%`; хост загружает каталоги с `/_plugins/<id>/locales/<code>.json`.

Предпросмотр установки показывает коды встроенных языков.

## Скрипт main.js

Типичные экспорты: `activate(ctx)`, `deactivate()`, `mount(el)`, `handleScheme(url)`.

При выданном `network.fetch`: `ctx.network.fetch()`. Проверяйте `ctx.capabilities.networkFetch` перед сетевой работой.

При WASM `backend`: `ctx.wasm.call("export", input)`. Строки через `ctx.i18n.t("key")`.

## Собранные WASM-модули

Файл `.wasm` может содержать манифест (`renbrowser.plugin`), файлы (`renbrowser.files`) и подпись (`renbrowser.signature`).

Установка: **Настройки → Расширения → Choose .wasm module**.

`extensions/micron-translator/`: TinyGo (`build-wasm.sh`, `go run ./extensions/micron-translator/bundle`).

## WASM backend

Поле `backend` указывает на WASM-модуль. При выданном `network.fetch` хост предоставляет `renhost.http_fetch`. Действуют лимиты запросов, таймауты и ограничения размера.

## DevTools

В **Developer tools → Network** исходящие HTTP-запросы расширений отображаются как **Extension fetch** со статусом и длительностью.

## Целостность

После установки RenBrowser сохраняет хеш файлов пакета. Изменения на диске вне приложения отключают расширение (**Изменено**). Повторное включение принимает текущее состояние.

## Безопасность

- Устанавливайте только из доверенных источников
- Читайте разрешения и endpoint перед подтверждением
- Предпочитайте подписанные расширения известных издателей
- Относитесь к плагинам как к локальным программам с доступом к профилю
