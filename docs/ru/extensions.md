# Расширения

Плагины добавляют URL-схемы, боковые панели, команды, темы, страницы настроек и вкладки DevTools.

## Установка

### Через настройки

1. **Настройки → Расширения**
2. **Установить из zip** или **из папки**
3. Проверьте манифест и разрешения
4. Включите расширение

### Вручную

Распакуйте в:

```
~/.renbrowser/plugins/<id>/
```

В каталоге должен быть `renbrowser.plugin.json`. `id` в манифесте лучше совпадает с именем папки.

## Пример

В репозитории: `extensions/hello-extension/`:

- схема `hello:`
- панель **Hello** в боковой панели
- команда **Say hello** с `mod+shift+h`

Используйте как шаблон.

## Файл манифеста

Имя: `renbrowser.plugin.json`

Обязательные поля:

| Поле | Назначение |
|------|------------|
| `manifestVersion` | Сейчас `1` |
| `id` | Уникальный id (3–128 символов: буквы, цифры, `.`, `-`) |
| `name` | Отображаемое имя |
| `version` | Строка semver |
| `main` | Точка входа фронтенда (опционально) |
| `permissions` | Список возможностей |

Опционально: `description`, `author`, `license`, `engines`, `backend`, `contributes`.

### Ограничение версии

```json
"engines": { "renbrowser": ">=0.2.0" }
```

Хост не загрузит плагин, если версия приложения слишком старая.

### Вклады (contributes)

| Тип | Назначение |
|------|------------|
| `urlSchemes` | Свои URL-схемы |
| `panels` | Панели в боковой панели и др. |
| `commands` | Команды и горячие клавиши |
| `themes` | JSON тем |
| `settings` | Подстраницы настроек |
| `devtools` | Вкладки DevTools |
| `renderers` | Рендереры для MIME или расширений |

## Разрешения

Плагин объявляет нужные возможности. Известные разрешения:

| Разрешение | Позволяет |
|------------|-----------|
| `storage.plugin` | Локальное хранилище плагина |
| `navigation.read` | Читать URL и вкладки |
| `navigation.write` | Инициировать навигацию |
| `network.fetch` | Сетевые запросы через разрешённые API |
| `events.emit` | Отправлять события хоста |
| `events.subscribe` | Подписываться на события |
| `devtools.network` | Доп. сеть в DevTools |
| `render.unsanitized` | Меньше санитизации HTML (опасно) |

Хост проверяет разрешения в runtime.

## Скрипт main.js

Типичные экспорты:

- `activate(ctx)` : подписки и UI
- `deactivate()` : очистка
- `mount(el)` : HTML боковой панели
- `handleScheme(url)` : обработчик схемы

См. hello-extension.

## WASM backend

Поле `backend` может указывать на WASM-модуль. WASM работает в ограниченной среде с явными правами.

## Безопасность

- Устанавливайте плагины только из доверенных источников
- Читайте список разрешений перед включением
- Относитесь к плагинам как к локальным программам с доступом к профилю
