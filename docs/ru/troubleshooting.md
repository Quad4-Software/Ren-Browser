# Решение проблем

Типичные сбои и что проверить первым делом.

## Reticulum не стартует

**Признаки:** в логе `reticulum start: ...`, пустое Обнаружение, все mesh-страницы падают.

**Проверки:**

1. Путь к конфигу на `about:` совпадает с реальным расположением файлов
2. `REN_BROWSER_CONFIG` или `--config` указывают на валидный файл
3. Синтаксис интерфейсов корректен
4. Пользователь процесса читает ключи и каталоги хранения

Исправьте конфиг Reticulum, затем перезагрузите из Настроек или перезапустите приложение.

## Пустое Обнаружение

См. [Обнаружение](discovery.md) и [Настройка Reticulum](reticulum-setup.md).

Кратко:

- подождите после подключения
- проверьте пиров на ваших интерфейсах
- файрвол для UDP или TCP портов из конфига

## Таймаут загрузки страницы

1. Проверьте хеш в адресной строке
2. Откройте другой узел из Обнаружения
3. Убедитесь, что в Настройках есть трафик на интерфейсах
4. Повторите после смены радио или маршрута в mesh

## База повреждена или не открывается

**Признаки:** ошибка профиля, предложение сбросить базу.

**Варианты:**

1. Восстановите `renbrowser.db` из резервной копии ([Данные и профили](data-and-profiles.md))
2. Сброс через UI (удаляет вкладки, историю, избранное, настройки)
3. Переименуйте повреждённый файл и дайте приложению создать новую базу

Идентичность Reticulum при сбросе браузера не затрагивается.

## Ошибка WASM или парсера Micron

При сбое SRI для Micron WASM:

1. Не отключайте проверку
2. Переустановите с официальных релизов
3. При сборке из исходников снова `task build` без ручной правки `frontend/dist/vendor/`

## Сервер: пустая страница или неверные ассеты

1. `--base-path` совпадает с mount прокси
2. `--trust-proxy` при TLS на прокси
3. Проброс порта в Docker (`-p 8080:8080`)

## Сервер: общая история, когда не хотели

Запускайте с `--public-mode`, чтобы данные были в `localStorage` каждого браузера.

## Расширение не загружается

1. Валидный JSON в `renbrowser.plugin.json`
2. `id` совпадает с именем папки в `plugins/`
3. Версия приложения удовлетворяет `engines.renbrowser`
4. Неизвестные строки в `permissions` блокируют загрузку

Текст ошибки в Настройках.

## Сбой сборки Android

1. `ANDROID_HOME`
2. `task android:install:deps`
3. API 34 и NDK r26+ ([Установка](installation.md))

## Разработка: падает `task check`

| Область | Команда |
|------|---------|
| Формат Go | `task fmt:go` |
| Тесты Go | `task test:go` |
| Фронтенд | `task frontend:check` |
| Безопасность | `task gosec` |

Перед отправкой патча: `task check`.

## Всё ещё не работает

1. Версия с `about:`
2. Логи терминала или Docker
3. Сообщество mesh или отчёт по каналам проекта
