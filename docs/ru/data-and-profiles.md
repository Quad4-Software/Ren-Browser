# Данные и профили

Закладки, история, вкладки, настройки и кэш обнаружения хранятся на диске.

## Расположение по умолчанию

| Элемент | Путь |
|------|------|
| Каталог данных | `~/.renbrowser/` |
| Основная база | `~/.renbrowser/renbrowser.db` |
| Плагины | `~/.renbrowser/plugins/<id>/` |
| Именованные профили | `~/.renbrowser/profiles/<name>/renbrowser.db` |
| Старый формат (миграция) | `~/.renbrowser/state.json` |

После обновления `state.json` один раз импортируется в SQLite при первом запуске.

## Что в SQLite

Обычно хранятся:

- открытые вкладки и сессия
- история с метками времени
- избранное
- настройки и горячие клавиши
- кэш обнаружения
- тема и пользовательские темы

При повреждении базы UI может предложить сброс. Сброс удаляет локальные вкладки, историю, избранное и настройки.

## Данные Reticulum отдельно

Ключи и конфиг интерфейсов в каталоге Reticulum (`~/.reticulum-go/` по умолчанию). Ren Browser читает этот путь, но не переносит идентичность Reticulum в `~/.renbrowser/`.

## Именованные профили

`--profile NAME` или `REN_BROWSER_PROFILE=NAME`:

```
~/.renbrowser/profiles/NAME/renbrowser.db
```

Отдельные истории на одном аккаунте (работа и личное, тесты).

## Импорт и экспорт

Только при старте:

- `--export-profile /path/to/backup.json` : записать и выйти
- `--import-profile /path/to/backup.json` : загрузить из файла

Переменные: `REN_BROWSER_EXPORT_PROFILE`, `REN_BROWSER_IMPORT_PROFILE`.

Делайте экспорт перед крупными обновлениями или переносом на другую машину.

## Хранение в серверном режиме

| Режим | Вкладки, история, избранное |
|------|------------------------------|
| Сервер по умолчанию | SQLite на сервере в `~/.renbrowser/` |
| `--public-mode` | `localStorage` каждого клиентского браузера |

## Темы

Экспорт и импорт JSON из Настроек. Это только оформление, не полный профиль.

## Данные плагинов

Расширения с `storage.plugin` получают изолированное хранилище по id. После удаления плагина папку `~/.renbrowser/plugins/<id>/` можно удалить вручную.

## Android

Та же схема в песочнице приложения. Пути другие, структура базы та же.

## Резервная копия

1. Закройте Ren Browser
2. Скопируйте `~/.renbrowser/renbrowser.db` (или путь профиля)
3. При необходимости скопируйте `~/.reticulum-go/`
4. Скопируйте `~/.renbrowser/plugins/` при использовании расширений

Восстановление: положите файлы на место до следующего запуска.
