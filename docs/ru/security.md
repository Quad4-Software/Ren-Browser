# Безопасность

Ren Browser рассчитан на системы и сети, которым вы доверяете. Кратко о безопасном использовании, проверке загрузок и отчётах.

## Границы доверия

| Поверхность | Риск |
|-------------|------|
| Настольное приложение | Локальный webview с Go-биндингами. Node.js в контенте страниц нет. |
| Серверный режим | Открытый HTTP-порт. Входа нет. |
| Плагины | Работают с объявленными разрешениями. WASM в песочнице. |
| Контент mesh | Как любой сетевой контент. HTML Micron санитизируется, кроме `render.unsanitized` у плагина. |

## Серверный режим

У `renbrowser-server` **нет аутентификации**. Публикация в интернет несёт риски:

- автоматическое сканирование
- злоупотребление интерфейсами Reticulum
- перегрузка однопроцессного приложения

Если всё же публикуете:

1. Обратный прокси с контролем доступа
2. HTTPS с валидным сертификатом
3. Ограничение порта файрволом или VPN
4. Актуальные обновления

См. [Серверный режим](server-mode.md).

## Плагины на desktop

Устанавливайте расширения только из доверенных источников. Читайте разрешения в **Настройки → Расширения**.

## Проверка загрузок

Официальные сборки: [GitHub Releases](https://github.com/Quad4-Software/Ren-Browser/releases) и CI GitHub Actions.

К релизу прилагается `SHA256SUMS.txt`:

```sh
sha256sum -c SHA256SUMS.txt
```

Для Docker лучше фиксировать digest (`@sha256:...`). Образы на GHCR с provenance и SBOM от Docker Buildx.

Несовпадение контрольных сумм : повод не доверять файлу.

## SRI для WASM

WASM-парсер Micron и `wasm_exec.js` проверяются SHA-384 SRI до запуска. Несовпадение хеша блокирует код и показывает ошибку.

## Данные на диске

- Состояние приложения: SQLite в `~/.renbrowser/`
- Ключи Reticulum: каталог конфига Reticulum
- Публичный сервер: часть данных только в `localStorage` клиентов

Шифруйте диск ОС на общих или переносных машинах.
