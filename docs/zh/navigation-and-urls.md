# 导航与 URL

Ren Browser 在地址栏中接受多种 URL 格式。本页列出了它们并解释规范化的工作方式。

## NomadNet 目标地址

完整的 NomadNet URL 形如：

```
<32个十六进制字符>:/page/path.mu
```

示例：

```
a1b2c3d4e5f6789012345678abcdef01:/page/index.mu
```

哈希值是以十六进制表示的 Reticulum 目标身份标识。`:/` 之后的路径是该 NomadNet 节点上的文件。

## 简写哈希值

如果只输入 32 字符的十六进制字符串，Ren Browser 会将其扩展为：

```
<hash>:/page/index.mu
```

这与常见的 NomadNet 主页路径相匹配。

## 不含哈希值的路径

如果当前上下文已有目标地址，以 `/page/` 开头的路径可能会相对于该节点解析。对于冷启动导航，优先使用完整的 `hash:/page/...` 格式。

## 内置方案

这些方案在应用内部处理，不使用网状网络。

| 方案 | 别名 | 描述 |
|------|------|------|
| `about:` | `about` | 版本、构建信息、Reticulum 配置路径、数据目录 |
| `license:` | `license` | 项目许可证（MIT） |
| `editor:` | `editor` | Micron 源码编辑器 |

匹配不区分大小写，末尾空格会被修剪。

## 扩展 URL 方案

已安装的扩展可以在 `renbrowser.plugin.json` 中注册自定义方案。例如，hello 扩展注册了 `hello:` 方案。参见[扩展](extensions.md)。

## 设置与内部路由

UI 可能使用内部路由来管理面板。地址栏专注于网状网络和内置方案。通过侧边栏按钮或 `Ctrl+,` / `Cmd+,` 打开**设置**。

## 标签页标题

标签页标题来源于：

1. 节点提供标题时的页面元数据
2. 哈希值匹配已知节点时，发现面板中的显示名称
3. 缩短的哈希值或路径作为备用

## 历史记录条目

每次加载内容的导航都可能创建一条包含 URL、标题、目标哈希值和时间戳的历史记录行。`about:` 等内置页面也包含在内。

## Micron 页面上的链接解析

点击渲染页面上的链接时：

- `about:`、`license:` 和 `editor:` 在本地打开
- 绝对网状网络 URL 直接导航
- 相对路径与当前页面目标地址组合

## 规范化规则（摘要）

| 你输入的内容 | 规范化后的 URL |
|-------------|---------------|
| `about` | `about:` |
| `license` | `license:` |
| `editor` | `editor:` |
| `abcdef...`（32 个十六进制字符） | `abcdef...:/page/index.mu` |
| 完整的 `hash:/page/foo.mu` | 不变 |
